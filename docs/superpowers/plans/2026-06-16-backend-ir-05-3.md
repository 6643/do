# Backend IR 05.3 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Extend the standalone backend IR with the smallest accessor-copy fold and trivial function inline regressions so later optimization work can stay off token-level WAT strings.

**Architecture:** Keep all work inside `tool/build/backend_ir.zig`. Add one module-level pass for trivial accessor-style copy folding and one module-level pass for trivial constant-return function inlining. Do not wire the IR into `codegen.zig` yet; this phase only proves the IR can host the optimizations as independent transforms.

**Tech Stack:** Zig compiler code, Zig unit tests, Markdown docs.

---

### Task 1: Lock the 05.3 regressions with failing tests

**Files:**
- Modify: `tool/build/backend_ir.zig`

- [ ] **Step 1: Add a failing test for accessor-style copy folding**

Add a test that builds a function with one block containing:

```zig
try block.instrs.append(allocator, .{ .local_get = v0 });
try block.instrs.append(allocator, .{ .local_set = v0 });
```

The test should assert that a new pass removes the redundant pair and leaves the block with no instructions.

- [ ] **Step 2: Add a failing test for trivial constant call inlining**

Add a test that builds a module with:

1. one callee function that has a single block with `const_i32 42` and `ret`,
2. one caller function whose entry block contains a single `call "callee"` instruction.

The test should assert that a new pass replaces the caller call instruction with `const_i32 42`.

- [ ] **Step 3: Run the backend IR tests and confirm they fail**

Run: `cd tool && zig test build/backend_ir.zig`
Expected: FAIL because the new pass helpers do not exist yet.

### Task 2: Implement the passes in the standalone IR

**Files:**
- Modify: `tool/build/backend_ir.zig`

- [ ] **Step 1: Add a function-level accessor-copy fold**

Implement a pass that removes consecutive `local_get` + `local_set` pairs when both operate on the same `ValueId`.

- [ ] **Step 2: Add a module-level trivial call inliner**

Implement a module container that can find callee functions by name and inline a callee only when:

1. it has exactly one block,
2. that block has exactly one `const_i32` instruction,
3. its terminator is `ret`.

Replace caller `call` instructions with the callee constant.

- [ ] **Step 3: Keep the module isolated**

Do not modify `tool/build/codegen.zig` in this phase. The IR module remains a standalone regression target.
