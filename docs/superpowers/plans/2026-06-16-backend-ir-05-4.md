# Backend IR 05.4 Evaluation Note

**Goal:** Decide whether the current backend phase should add a direct wasm binary emitter, or explicitly keep WAT text as the compiler output boundary for now.

**Conclusion:** Keep WAT text output for this phase. Do not implement a direct wasm binary emitter yet.

**Evidence:**

1. `tool/build/run.zig` still writes the output of `codegen.emitWatWithOptions(...)` and `codegen.emitTestWat(...)` directly to the requested path.
2. `tool/build/codegen.zig` still builds the whole module through `std.ArrayList(u8)` plus `appendSlice(...)`/`appendFmt(...)`, which means the backend contract is currently token-level WAT text, not a typed binary encoder surface.
3. `tool/build/test/run_tests.sh` and `tool/build/test/README.md` treat `.wat` as the canonical compiler artifact, then call `wasm-tools parse` to derive `.wasm` for execution and validation.
4. Existing WASI/component validation tools also consume generated WAT text or WAT fragments such as `core.wat`, `core_component.wat`, `core_imports.wat`, and `core_shims.wat`.

**Why not implement now:**

- A binary emitter would not be a local backend optimization pass; it would change the compiler's public output boundary, CLI defaults, and test harness assumptions.
- The new standalone `backend_ir.zig` is still intentionally isolated and is not yet the source of truth for full code generation.
- Adding a binary writer now would either duplicate the entire existing WAT lowering logic or force an unscheduled codegen rewrite.

**Recommended recovery point:**

Treat direct wasm binary emission as a later dedicated task only if at least one of these becomes true:

1. WAT text generation or `wasm-tools parse` becomes a measurable performance bottleneck.
2. The backend IR becomes the primary lowering boundary for full modules.
3. The CLI needs first-class `.wasm` output instead of `.wat`.

**Current next step after 05.4:**

Stay on the roadmap without wiring `backend_ir.zig` into `codegen.zig` yet. The next backend task should be chosen as another isolated IR-side optimization or a later explicit codegen rewrite milestone, not a partial binary emitter.
