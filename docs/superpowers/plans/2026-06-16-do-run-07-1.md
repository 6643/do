> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a first-version `do run <input.do>` command that executes the current core wasm smoke subset through the existing `build -> wasm-tools parse -> node` bridge.

**Architecture:** Keep the new command narrowly scoped. Reuse the existing compiler frontend and WAT codegen path, add a dedicated `tool/run/run.zig` command module for orchestration, and add a product-facing JS runner under `tool/run/` rather than reusing test-only script paths directly. Validation stays aligned with the existing `tool/build/test/run/*.do` smoke cases plus new CLI error-path checks.

**Tech Stack:** Zig compiler code, Node.js runner script, shell regression harness, Markdown docs.

- [x] **Step 3: Implement `parseRun(args)` in `tool/build/cli.zig`**

Add a dedicated parser that only accepts one positional input path and no output or mode flags:

```zig
pub const RunArgs = struct {
    input_path: []const u8,
};

pub fn parseRun(args: []const []const u8) !RunArgs {
    var input_path: ?[]const u8 = null;
    var i: usize = 1;
    while (i < args.len) : (i += 1) {
        if (std.mem.startsWith(u8, args[i], "-")) return error.UnexpectedCliArg;
        if (input_path != null) return error.UnexpectedCliArg;
        input_path = args[i];
    }
    return .{ .input_path = input_path orelse return error.MissingInputPath };
}
```

- [x] **Step 1: Copy the current smoke-run behavior into a product-facing script**

Create `tool/run/run_wasm_program.mjs` with the same current runtime contract as the smoke script:

```js
import { readFileSync } from "node:fs";

const wasmPath = process.argv[2];
if (!wasmPath) {
  console.error("usage: node run_wasm_program.mjs <program.wasm>");
  process.exit(2);
}

const bytes = readFileSync(wasmPath);
const logs = [];
const decoder = new TextDecoder("utf-8", { fatal: true });
let instance;

const imports = {
  env: {
    add(a, b) { return a + b; },
    dep_add(a, b) { return a + b; },
    log(ptr, len) {
      const memory = instance?.exports?.memory;
      if (!(memory instanceof WebAssembly.Memory)) {
        throw new Error("missing exported memory");
      }
      const view = new Uint8Array(memory.buffer, ptr, len);
      logs.push(decoder.decode(view));
    },
  },
};

const loaded = await WebAssembly.instantiate(bytes, imports);
instance = loaded.instance;

if (typeof instance.exports._start !== "function") {
  throw new Error("missing _start export");
}

instance.exports._start();

for (const line of logs) {
  console.log(line);
}
```

The real flow should:

1. parse `run` args,
2. locate `wasm-tools`,
3. locate `node`,
4. compile input to WAT using the shared build helper,
5. create temp dir,
6. write `out.wat`,
7. run `wasm-tools parse out.wat -o out.wasm`,
8. run `node tool/run/run_wasm_program.mjs out.wasm`,
9. forward stdout/stderr,
10. return the child exit status.

Use `std.process.Child.run` or the closest Zig 0.16 equivalent rather than shelling out through `sh -c`.
