# Backend IR 05.1 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Define the smallest stable backend IR input/output boundary so later backend optimizations do not operate directly on token-level WAT strings.

**Architecture:** Introduce a standalone `tool/build/backend_ir.zig` that models backend functions, blocks, instructions, and terminators without changing current WAT emission. Keep this phase documentation-first and test-first: the IR module is self-contained, unit-tested, and not yet wired into `codegen.zig`.

**Tech Stack:** Zig compiler code, Zig unit tests, Markdown docs.
